WKWebView Migration Notes
=========================

Goal: move GitX web views from legacy WebView/WebScript APIs to a WKWebView-friendly bridge based on JSON messaging.

Current state:
- `gitxBridge` provides queued post/subscribe helpers; commit and history pages use it exclusively.
- `PBWebController` only exposes `postJSONMessage:` to JavaScript and flushes queued bridge messages on bootstrap.
- Remaining WebView clients: `PBWebChangesController`, `PBWebHistoryController`, and the base `PBWebController` (via `PBWebViewBridge`).

Next steps:
- Port the remaining controllers to a WKWebView-backed bridge (replace `callWebScriptMethod`, expose WK message handlers).
- Remove the legacy `Controller` injection once all views use `gitxBridge` exclusively.
- Prototype a WKWebView wrapper that reuses the new bridge contracts and start retiring WebView-specific code.
